## Network Dependence, Regularization, and Mean-Field Bias in Agent-Based SIR Models

## Purpose

Build a full research/demo repository for a CSSSA extended abstract and 20-minute live presentation. The project demonstrates that fitting a mean-field ODE SIR model to coarse-grained data generated by a network-dependent agent-based SIR process can produce biased parameter inference. More data at the same observational grain can shrink posterior variance while leaving the posterior centered on the wrong, pseudo-true ODE parameter.

3. Julia inference code for fitting an ODE SIR model to coarse aggregate data.
4. Figures showing posterior bias and variance shrinkage without bias reduction.
5. A regularization-path demonstration showing a community network collapsing toward the uniform graph.
6. A grain/clock demonstration showing how temporal coarsening hides event ordering and network dependence.
7. A Reveal.js slide deck with SVG animations driven by Julia over WebSockets.
8. Static fallback figures or animations for every live slide.
9. Clear README and reproducible run commands.

## Core Argument

Agent-based models are often criticized for being too flexible and difficult to validate. This project reframes ABMs as generative statistical models. The simulator produces a latent social process, while data are generated through an observation process at a chosen level of aggregation and time scale.

Using SIR as the running example, the project shows that the standard ODE SIR model is not a neutral baseline. It is the limiting case obtained when network structure, individual heterogeneity, and event ordering are smoothed or regularized away.

The standard ODE incidence term assumes that susceptible-infected contact opportunities are proportional to the aggregate product \(SI\):

```math
\frac{dI}{dt} = \beta SI - \gamma I.
```

But in a network SIR process, incidence depends on susceptible-infected edges:

```math
\text{incidence}_{network}(t) = \beta E_{SI}(t).
```

The ODE approximation assumes:

```math
E_{SI}(t) \approx c S(t)I(t).
```

When the true process has community structure or other strong network dependence, this approximation fails. A fitted ODE transmission parameter estimates an effective aggregate rate, not necessarily the underlying microscopic transmission parameter.

The demonstration should show:

```math
\beta^*_{ODE} \neq \beta_{true},
```

where \(\beta^*_{ODE}\) is the pseudo-true parameter around which the misspecified ODE posterior concentrates.

As more data are collected at the same coarse grain:

```math
p(\beta_{ODE}\mid Y_{coarse}, n) \to \delta_{\beta^*_{ODE}},
```

not

```math
\delta_{\beta_{true}}.
```

### Extended Abstract Required Structure

#### 1. Problem

Agent-based models represent heterogeneous agents, network structure, local interaction, and path dependence, but they are often criticized for weak validation, excessive flexibility, and unclear relation to standard aggregate models.

Opening line candidate:

> Agent-based models are powerful because they can represent heterogeneous agents, local interaction, network structure, and path dependence. They are controversial for the same reason: the flexibility that makes them expressive can also make them difficult to validate.

#### 2. Contribution

Reframe ABMs as generative statistical models:

```text
simulator -> latent social process -> observation process -> data
```

Key claim:

> A simulation is not yet a statistical model until it specifies how latent simulated states become observed data.

#### 3. SIR Example

Use SIR because it is familiar and has a clear mean-field limiting form.

Explain that the product term \(SI\) is a social assumption, not merely algebra.

Key line:

> The term \(SI\) encodes a uniform-mixing assumption: susceptible-infected contact opportunities are assumed proportional to the product of aggregate susceptible and infected counts.

#### 4. Network SIR

Introduce the network process:

```math
P(X_i(t+\Delta t)=I \mid X_i(t)=S) = 1 - \exp\left(-\beta \Delta t \sum_j A_{ij}\mathbf{1}\{X_j(t)=I\}\right).
```

Where \(A\) is a contact network.

Explain that in the network model, incidence depends on \(E_{SI}(t)\), the number of susceptible-infected edges.

#### 5. Regularization

Explain regularization as penalizing unnecessary model flexibility.

For agent-level heterogeneity:

```math
\beta_i = \bar{\beta} + \delta_i,
\qquad
\lambda \frac{1}{N}\sum_i \delta_i^2.
```

Explain the normalization by \(N\): the penalty must mean the same thing as the number of agents changes.

For network regularization, define a simple generator:

```math
G \sim \mathcal{G}_{\eta},
\qquad
\eta = 0 \Rightarrow \text{uniform mixing}.
```

Regularize:

```math
\lambda \eta^2.
```

Key line:

> The uniform graph is not the neutral case; it is the maximally regularized case.

#### 6. Coarsening and Mean-Field Limit

Explain that the ODE is reached when three kinds of information are removed:

1. Agent states are aggregated into counts.
2. Network structure is regularized toward uniform mixing.
3. Event ordering is smoothed by coarse time aggregation.

Use this line:

> The ODE is not what happens when an ABM gets large. It is what happens when the ABM gets large, coarse, and socially uniform.

#### 7. Flagship Simulation Result

Describe simulation experiment:

1. Generate data from a network SIR ABM with known \(\beta_{true}\).
2. Observe only coarse weekly aggregate counts.
3. Fit a mean-field ODE SIR model to those counts.
4. Plot posterior for \(\beta_{ODE}\).
5. Show the true \(\beta_{true}\) is off-center or outside the posterior region.
6. Increase data at the same grain and show posterior variance shrinks while bias remains.

Key line:

> More data at the same observational grain do not recover the missing mechanism; they estimate the wrong reduced model more precisely.

#### 8. PAC-Bayes and Perturbation Stability

Mention briefly and accessibly.

Do not make this the centerpiece.

Suggested wording:

> This argument connects to PAC-Bayesian ideas, which formalize the tradeoff between empirical fit and the amount of model flexibility used to obtain that fit. In this setting, the prior represents a distribution over plausible mechanisms before calibration, while the fitted model is penalized for moving too far from those commitments. More generally, regularization can be understood as a requirement that conclusions remain stable under modeling changes that the theory regards as scientifically irrelevant.

#### 9. Model Criticism

Emphasize practical checks:

- compare against mean-field limit;
- vary observation grain and clock;
- vary network regularization strength;
- inspect posterior bias;
- run posterior predictive checks;
- test robustness to network realizations.

#### 10. Conclusion

End with:

> The broader goal is to make social simulation empirically disciplined: not merely to generate plausible patterns, but to determine which mechanisms are needed, which can be regularized away, and where a model’s claims cease to be robust.

## Network Generators

Implement two network generators.

### 1. Uniform Graph

Erdős–Rényi baseline:

```math
P(A_{ij}=1)=p_0.
```

Function:

```julia
generate_uniform_graph(N::Int, p::Float64, rng)::SimpleGraph
```

### 2. Community Graph with Regularization Parameter

Simple stochastic block model:

```math
\operatorname{logit}P(A_{ij}=1)=\alpha + \eta B_{ij}.
```

Where:

```math
B_{ij}=1 \quad \text{if same group},
```

and

```math
B_{ij}=-b \quad \text{if different group}.
```

Choose \(b\) so expected density stays approximately stable as \(\eta\) changes. For simplicity, start with:

```math
b = \frac{1}{K-1}
```

where \(K\) is number of groups.

When \(\eta=0\), all dyads have probability \(\sigma(\alpha)\), so the graph is uniform.

Functions:

```julia
logistic(x) = 1 / (1 + exp(-x))

generate_community_graph(pop::Population, params::NetworkGeneratorParams, rng)::SimpleGraph
```

Regularization penalty:

```julia
network_regularization(params::NetworkGeneratorParams, lambda_eta::Float64) = lambda_eta * params.eta^2
```

## Network SIR Simulation

Discrete-time network SIR:

For susceptible agent \(i\):

```math
P(i \text{ infected in } [t,t+dt]) = 1 - \exp(-\beta dt \cdot n_i^I(t)),
```

where \(n_i^I(t)\) is the number of infected neighbors.

For infected agent \(i\):

```math
P(i \text{ recovers in } [t,t+dt]) = 1 - \exp(-\gamma dt).
```

Use synchronous update by default. Later allow async toggle for live demo.

Function:

```julia
simulate_network_sir(
    g::SimpleGraph,
    params::SIRParams;
    T::Float64,
    dt::Float64,
    initial_infected::Vector{Int},
    rng
)::DataFrame
```

Return DataFrame columns:

```text
t, S, I, R, new_infections, new_recoveries
```

Also optionally return agent states for animation:

```julia
simulate_network_sir_frames(...)
```

which returns records with:

```text
t, agent_id, x, y, group, status
```

Use a simple layout for visualization:

- group-based circular layout;
- or random coordinates clustered by group.

No need for force-directed layout v1.

# Deliverable 3: ODE SIR and Grid Posterior

## ODE SIR Model

Implement:

```julia
solve_ode_sir(params::SIRParams, N::Int, I0::Float64, R0::Float64, T::Float64; saveat)
```

Use DifferentialEquations.jl.

ODE:

```math
\frac{dS}{dt} = -\beta_{ODE}\frac{SI}{N}
```

```math
\frac{dI}{dt} = \beta_{ODE}\frac{SI}{N} - \gamma I
```

```math
\frac{dR}{dt} = \gamma I
```

Use consistent parameterization with network model. Be explicit in comments: network \(\beta\) is per infected-neighbor contact rate; ODE \(\beta_{ODE}\) is an effective mass-action rate. They are not automatically the same.

Weekly predicted incidence:

```math
\widehat{Y}_k = S(t_k) - S(t_{k+1}).
```

## Grid Posterior

Fit ODE to coarse weekly observations using grid posterior over \(\beta_{ODE}\) and optionally \(\gamma\).

Recommended v1:

- fix \(\gamma\) at true value for the clearest \(\beta\) bias demonstration;
- then add optional two-dimensional grid over \((\beta,\gamma)\).

This isolates the core result.

Observation likelihood:

Use transformed Gaussian:

```math
\log(1+Y_k) \sim \mathcal{N}(\log(1+\widehat{Y}_k), \sigma^2).
```

Function:

```julia
loglik_ode_beta(beta, gamma, obs_df, N, I0, R0, sigma)
```

Prior:

Use weakly informative prior over \(\beta\):

```math
\beta \sim \text{LogNormal}(m, s)
```

or uniform over a reasonable grid for simplicity.

Function:

```julia
grid_posterior_beta(beta_grid, gamma, obs_df, N, I0, R0, sigma)
```

Return:

```text
beta, loglik, logprior, logpost, posterior_weight
```

Normalize with log-sum-exp.

## Posterior Bias Demonstration

Generate data from community network SIR with known:

```text
N = 1000
K = 4 groups
beta_true = choose by simulation calibration
 gamma_true = choose by simulation calibration
eta_true = moderate/strong community structure
T = enough duration to observe outbreak
```

Then fit ODE to weekly observations.

Create posterior plots:

1. Single posterior showing true value off-center.
2. Three posterior curves with increasing data amount at same grain:
   - one outbreak;
   - five independent outbreaks;
   - twenty independent outbreaks.

For repeated outbreaks, the likelihood is product across independent simulated datasets:

```math
\log L(\beta) = \sum_{r=1}^R \log L_r(\beta).
```

Important: keep the observation grain the same. Only increase repeated coarse data.

Expected visual:

- posterior variance shrinks;
- posterior center remains away from \(\beta_{true}\);
- pseudo-true value \(\beta^*_{ODE}\) is marked.

Pseudo-true estimate:

```julia
beta_star = beta_grid[argmax(loglik_average_or_posterior)]
```

Figure caption:

> More coarse-grained data reduce posterior variance around the ODE’s pseudo-true parameter, but do not remove bias caused by fitting the wrong interaction structure.

# Notes on Parameterization

Be explicit in code comments and paper text:

- Network \(\beta_{true}\) is a per infected-neighbor transmission rate.
- ODE \(\beta_{ODE}\) is an effective aggregate mass-action rate.
- They are not automatically on the same scale.
- For the figure, choose scaling so the comparison is interpretable and explained.

One option is to compare the ODE posterior against the ODE value implied by the uniform-graph case with the same microscopic beta. In that case, define:

```math
\beta_{mf,true} = \beta_{true} \bar d.
```

Then the posterior plot marks:

- uniform-graph effective beta: \(\beta_{mf,true}\);
- ODE pseudo-true beta under community data: \(\beta^*_{ODE}\).

This may be more interpretable than comparing raw microscopic \(\beta\) directly to ODE \(\beta\).

The abstract should say this clearly:

> The fitted ODE parameter is compared to the effective mean-field value implied by the same microscopic transmission rate under uniform mixing. Under community structure, the posterior concentrates away from that value.

This avoids unit confusion.

# Draft Extended Abstract Text

Claude Code should create a polished 1000–2000 word version based on the following draft. Revise for flow, but preserve the argument.

```markdown
# When More Data Make the Wrong Model More Certain
## Network Dependence, Regularization, and Mean-Field Bias in Agent-Based SIR Models

Agent-based models are powerful because they can represent heterogeneous agents, local interaction, network structure, and path dependence. They are controversial for the same reason: the flexibility that makes them expressive can also make them difficult to validate. A sufficiently flexible simulation may reproduce a pattern without making clear which mechanisms are responsible, how the simulation is connected to data, or how its conclusions would change under alternative assumptions.

This paper proposes a statistical framing of agent-based models as generative models. In this view, a simulator produces a latent social process, while observed data arise through a separate observation process operating at a specified level of aggregation and time scale. The distinction is important because many social data sets do not observe the process of interest directly. They record institutional traces, aggregate counts, sampled events, or temporally coarsened measurements. A simulation is therefore not yet a statistical model until it specifies how latent simulated states become observed data.

The argument is developed using epidemic SIR models. The familiar mean-field ordinary differential equation assumes that susceptible-infected contact opportunities are proportional to the product of aggregate susceptible and infected counts. The term \(SI\) is therefore not merely algebraic; it encodes a strong social assumption about contact structure. A network SIR model relaxes this assumption by allowing infection to travel through a contact network. In the network model, incidence depends on the number of susceptible-infected edges, not directly on the aggregate product \(SI\). The mean-field model is appropriate only when the relevant contact structure is sufficiently homogeneous that susceptible-infected edges are well approximated by a function of aggregate counts.

The paper then introduces regularization as a way to make this relationship between aggregate and agent-based models explicit. By regularization I mean the practice of penalizing unnecessary model flexibility. If each agent has an individual infection parameter, those individual deviations should be shrunk toward a population mean unless the data justify the added heterogeneity. Similarly, if the contact network is unobserved, the model should not estimate every possible edge. Instead, the network should be generated by a simpler process whose deviations from uniform mixing are penalized. In the simple network generator used here, a single parameter controls the strength of community structure. When that parameter is zero, the generator reduces to a uniform random contact structure. In this sense, the uniform graph is not the neutral case; it is the maximally regularized case.

This framing also clarifies the relation between ABMs and standard aggregate models. The standard SIR ODE is not what happens simply when an ABM becomes large. Rather, it is what happens when agent states are aggregated into counts, network structure is regularized toward uniform mixing, event ordering is smoothed by coarse time aggregation, and a large-population deterministic limit is taken. The ODE is therefore a limiting case of a more general generative model, not a rival paradigm.

The central demonstration shows why this matters for inference. Data are generated from a network SIR model with known transmission parameters and strong community structure, but the analyst observes only coarse aggregate incidence counts. A mean-field ODE SIR model is then calibrated to those observations. The fitted ODE may track the aggregate curve while producing a posterior distribution for the transmission parameter that is centered away from the value implied by the true microscopic process under uniform mixing. The ODE is not learning the underlying transmission mechanism; it is learning an effective aggregate rate that absorbs network structure, temporal aggregation, and observation scale.

The most important result is that increasing the amount of data at the same observational grain reduces posterior variance but does not remove this bias. With more repeated outbreaks or longer aggregate series, the posterior becomes more concentrated around the ODE's pseudo-true parameter. The analyst becomes more certain, but about the wrong reduced model. More data at the same grain do not recover the missing mechanism; they estimate the misspecified aggregate model more precisely.

This result motivates a broader workflow for statistically accountable social simulation. First, specify the latent process and the observation process separately. Second, compare richer simulations against their regularized mean-field limits. Third, penalize unnecessary heterogeneity in agents and networks. Fourth, test whether conclusions are robust to changes in aggregation level, observation clock, and network realization. Finally, use posterior predictive checks and sensitivity analysis to identify where the model's claims stop.

The argument also connects to PAC-Bayesian ideas, which provide one way to formalize the cost of model flexibility. In this setting, prior assumptions define a distribution over plausible mechanisms before calibration, while fitted models are penalized for moving too far from those commitments in order to fit the data. More generally, regularization can be understood as a demand for stability under modeling changes that the theory regards as scientifically irrelevant.

The broader goal is to make social simulation empirically disciplined: not merely to generate plausible patterns, but to determine which mechanisms are needed, which can be regularized away, and where a model's claims cease to be robust.
```
